
按这个顺序做，成功率最高（你已经删掉旧 `extracted.json`，很好）：

## 1) 先用“分步跑”定位，不直接跑 `e2e`

在项目根目录 PowerShell：

```powershell
# 激活你之前运行成功的同一个环境（日志里是 oneLCA-TEA_pg）
conda activate oneLCA-TEA_pg

# 当前会话临时参数（先不改文件）
$env:LLM_TIMEOUT="900"
$env:EMBEDDING_TIMEOUT="120"
$env:MAX_ASYNC="1"
$env:LLM_TEMPERATURE="0.0"
$env:LLM_CONTEXT_LENGTH="32768"
$env:LLM_JSON_MODE="1"
```

```powershell
# 清理可能残留的 LightRAG 索引状态
Remove-Item -Recurse -Force ".\data\processed\lightrag_db" -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force ".\data\processed\knowledge_graph" -ErrorAction SilentlyContinue
```

```powershell
# Step 2: 先看索引是否还会超时
python scripts/build_knowledge_graph.py --step index --parser mineru --engine lightrag --limit 1

# Step 3: 再做抽取
python scripts/build_knowledge_graph.py --step extract --parser mineru --engine lightrag --limit 1

# Step 4: 最后构图
python scripts/build_knowledge_graph.py --step build --parser mineru --engine lightrag --limit 1
```

---

## 2) 立刻验证“是否真的修好”

```powershell
python -c "import json,glob; p=glob.glob(r'data/processed/extracted_data/*phosphogypsum*_extracted.json')[0]; d=json.load(open(p,encoding='utf-8')); print('file=',p); print('keys=',list(d.keys())); print('non_empty=',{k: bool(v) for k,v in d.items()})"
```

你要看到的不是 `{}`，而是至少有部分 `composition/technology/lci/cost/policy` 非空。

---

## 3) 如果仍出现 `Request timed out`

先做 Ollama 侧稳定化（系统级，执行后要重启 Ollama）：

```powershell
setx OLLAMA_FLASH_ATTENTION 0
setx OLLAMA_CONTEXT_LENGTH 32768
setx OLLAMA_NUM_PARALLEL 1
setx OLLAMA_MAX_LOADED_MODELS 1
```

然后：
- 彻底退出 Ollama（托盘 `Quit`），重新启动
- 重新开一个 PowerShell（让 `setx` 生效）
- 再执行上面的第 1、2 步

---

## 4) 最后再回到 smoke test

分步都通过后再跑：

```powershell
python examples/e2e_smoke_test.py --limit 1 --skip-raganything
```
